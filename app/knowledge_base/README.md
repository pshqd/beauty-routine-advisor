# Knowledge Base (Markdown) — правила оформления

Эта папка содержит **базу знаний** в формате Markdown для RAG-поиска (Retrieval Augmented Generation).
Чтобы семантический поиск работал качественно, тексты должны быть **структурированными** и **предсказуемыми**: заголовки → секции → списки/таблицы.

---

## 1) Общие правила Markdown

- Используем только **ATX-заголовки** (`#`, `##`, `###`), без Setext (`===`, `---`).
- В одном файле — **ровно один** заголовок уровня `#` (название документа).
- Не пропускаем уровни заголовков: `#` → `##` → `###` (не делаем `#` → `###`).
- Между заголовком и текстом — **пустая строка** (и до заголовка тоже желательно).
- Глубина заголовков: максимум до `###` (редко `####`, только если очень нужно).
- Заголовки должны быть **короткими и однозначными** (их будут видеть отдельно от контекста).

---

## 2) Как мы режем текст на чанки (важно для RAG)

Мы стараемся, чтобы каждая секция `##` была **самодостаточной**: внутри — один смысловой блок и практические рекомендации.

Рекомендации:

- Одна секция `##` = одна тема (например, «Жирная кожа: базовый уход»).
- Внутри секции используйте `###` для подподтем («Утро», «Вечер», «Чего избегать»).
- Не смешивайте в одной секции разные проблемы (например, акне + пигментация) — лучше разделить. [web:155]

---

## 3) Формат секции (шаблон)

Используйте этот шаблон для большинства разделов:

## <Тема секции>

Короткое описание в 1–2 предложения (что это и кому подходит).

### Симптомы / признаки

- ...
- ...

### Рекомендации

- Утро: ...
- Вечер: ...
- Частота: ...

### Ингредиенты (активы)

- Полезно: ...
- С осторожностью: ...
- Не рекомендуется: ...

### Ошибки и противопоказания

- ...
- ...

### Когда к дерматологу

- ...
- ...

---

## 4) Списки, таблицы, код

- Списки оформляем через `-` (один пункт = одна мысль).
- Таблицы допускаются, если они короткие и реально помогают (например, «ингредиент → эффект → кому подходит»).
- Кодовые блоки используем редко (обычно не нужны), только если это прям «рецепт» процедуры/шага.

---

## 5) Термины и стиль

- Пишем **простым языком**, без сложной медицинской терминологии (или объясняем термины в скобках).
- Не обещаем «лечит навсегда»; используем аккуратные формулировки: «может помочь», «часто используют».
- Если есть риск (сильное воспаление, подозрение на дерматит и т.д.) — добавляем секцию «Когда к дерматологу».

---

## 6) Запрещено / нежелательно

- Не пишем личные данные (телефоны, адреса, ФИО).
- Не вставляем длинные простыни текста без заголовков.
- Не делаем «всё обо всём» в одном файле.
- Не используем картинки как единственный носитель информации (RAG их не прочитает).

---

## 7) Именование файлов

Файлы называем в `snake_case`:

- `skin_types.md`
- `acne_treatment.md`
- `anti_aging.md`
- `nutrition.md`
- `routines.md`

Один файл — одна большая тема.

---

## 8) Минимальный пример правильного файла

```text
    # Acne treatment

    ## Что такое акне

    1–2 предложения определения.

    ### Признаки
    - Комедоны
    - Воспаления

    ## Базовая рутина при акне

    ### Утро
    - Очищение: ...
    - SPF: ...

    ### Вечер
    - Очищение: ...
    - Актив: ...

    ### Когда к дерматологу
    - Болезненные кисты
    - Быстрое ухудшение

```

## 9) Чек-лист перед PR

Перед тем как пушить изменения:

- [ ] В файле один `#` заголовок и он первый.
- [ ] Все темы разбиты на секции `##`.
- [ ] Есть пустые строки вокруг заголовков.
- [ ] Нет «огромной» секции на 200+ строк без подзаголовков `###`.
- [ ] Формулировки конкретные и практичные (утро/вечер/частота).
