# Knowledge Base (Markdown) — правила оформления

Эта папка содержит **базу знаний** в формате Markdown для RAG-поиска (Retrieval Augmented Generation).
Чтобы семантический поиск работал качественно, тексты должны быть **структурированными** и **предсказуемыми**: YAML-шапка → заголовки → секции → списки/таблицы.

---

## 1) YAML Frontmatter (обязательно)

Каждый файл должен начинаться с YAML-блока **до** заголовка `#`. Это метаданные для фильтрации при RAG-поиске.

### Обязательные поля

```yaml
---
title: "Читаемое название файла"
category: "раздел базы знаний"
tags: ["тег1", "тег2", "тег3"]
---
```

### Полный шаблон

```yaml
---
title: "Ретинол и его производные"
category: "косметика"
subcategory: "активные_ингредиенты"
tags: ["ретинол", "антиэйдж", "акне"]
skin_type: ["жирная", "комбинированная", "зрелая"]
skin_concern: ["морщины", "пигментация", "поры"]
season: ""              # весна | лето | осень | зима | все (если применимо)
procedure_type: ""      # домашний | салонный | оба (если применимо)
---
```

### Доступные значения полей

| Поле | Допустимые значения |
|---|---|
| `category` | `косметика`, `питание`, `уход_за_кожей`, `уход_за_волосами`, `уход_за_телом`, `процедуры`, `здоровье` |
| `skin_type` | `сухая`, `жирная`, `комбинированная`, `чувствительная`, `нормальная`, `зрелая`, `все типы` |
| `skin_concern` | `акне`, `постакне`, `морщины`, `пигментация`, `поры`, `купероз`, `розацеа`, `сухость`, `обезвоживание`, `тусклость` |
| `season` | `весна`, `лето`, `осень`, `зима`, `все` |
| `procedure_type` | `домашний`, `салонный`, `оба` |

### Правила

- YAML-блок — **первое**, что есть в файле (до `#` заголовка).
- Поля `title`, `category`, `tags` — **обязательны**.
- Остальные поля — заполняем только если применимы к теме файла (пустые строки не оставляем).
- `tags` — не более **6 тегов**; предпочитаем конкретные слова из словаря, а не новые.

---

## 2) Общие правила Markdown

- Используем только **ATX-заголовки** (`#`, `##`, `###`), без Setext (`===`, `---`).
- В одном файле — **ровно один** заголовок уровня `#` (название документа).
- Не пропускаем уровни заголовков: `#` → `##` → `###` (не делаем `#` → `###`).
- Между заголовком и текстом — **пустая строка** (и до заголовка тоже).
- Глубина заголовков: максимум до `###` (редко `####`, только если очень нужно).
- Заголовки должны быть **короткими и однозначными** (их будут видеть отдельно от контекста).

---

## 3) Как мы режем текст на чанки (важно для RAG)

Мы стараемся, чтобы каждая секция `##` была **самодостаточной**: внутри — один смысловой блок и практические рекомендации.

Рекомендации:

- Одна секция `##` = одна тема (например, «Жирная кожа: базовый уход»).
- Внутри секции используйте `###` для подподтем («Утро», «Вечер», «Чего избегать»).
- Не смешивайте в одной секции разные проблемы (например, акне + пигментация) — лучше разделить.

---

## 4) Формат секции (шаблон)

Используйте этот шаблон для большинства разделов:

```
## <Тема секции>

Короткое описание в 1–2 предложения (что это и кому подходит).

### Симптомы / признаки

- ...
- ...

### Рекомендации

- Утро: ...
- Вечер: ...
- Частота: ...

### Ингредиенты (активы)

- Полезно: ...
- С осторожностью: ...
- Не рекомендуется: ...

### Ошибки и противопоказания

- ...
- ...

### Когда к дерматологу

- ...
- ...
```

---

## 5) Списки, таблицы, код

- Списки оформляем через `-` (один пункт = одна мысль).
- Таблицы допускаются, если они короткие и реально помогают (например, «ингредиент → эффект → кому подходит»).
- Кодовые блоки используем редко, только если это прям «рецепт» процедуры/шага.

---

## 6) Термины и стиль

- Пишем **простым языком**, без сложной медицинской терминологии (или объясняем термины в скобках).
- Не обещаем «лечит навсегда»; используем аккуратные формулировки: «может помочь», «часто используют».
- Если есть риск (сильное воспаление, подозрение на дерматит и т.д.) — добавляем секцию «Когда к дерматологу».

---

## 7) Запрещено / нежелательно

- Не пишем личные данные (телефоны, адреса, ФИО).
- Не вставляем длинные простыни текста без заголовков.
- Не делаем «всё обо всём» в одном файле.
- Не используем картинки как единственный носитель информации (RAG их не прочитает).

---

## 8) Именование файлов

Файлы называем в `snake_case`:

- `skin_types.md`
- `acne_treatment.md`
- `anti_aging.md`
- `nutrition.md`
- `routines.md`

Один файл — одна большая тема.

---

## 9) Минимальный пример правильного файла

```text
---
title: "Лечение акне"
category: "уход_за_кожей"
subcategory: "проблемы_кожи"
tags: ["акне", "воспаления", "комедоны", "салициловая_кислота"]
skin_type: ["жирная", "комбинированная"]
skin_concern: ["акне", "поры"]
---

# Acne treatment

## Что такое акне

1–2 предложения определения.

### Признаки
- Комедоны
- Воспаления

## Базовая рутина при акне

### Утро
- Очищение: ...
- SPF: ...

### Вечер
- Очищение: ...
- Актив: ...

### Когда к дерматологу
- Болезненные кисты
- Быстрое ухудшение
```

## 10) Чек-лист перед PR

Перед тем как пушить изменения:

- [ ] Файл начинается с YAML-блока (`---` ... `---`).
- [ ] В YAML заполнены `title`, `category`, `tags`.
- [ ] В файле один `#` заголовок и он идёт сразу после YAML.
- [ ] Все темы разбиты на секции `##`.
- [ ] Есть пустые строки вокруг заголовков.
- [ ] Нет «огромной» секции на 200+ строк без подзаголовков `###`.
- [ ] Формулировки конкретные и практичные (утро/вечер/частота).
- [ ] Теги из допустимого словаря (см. раздел 1).
