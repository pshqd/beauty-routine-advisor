# User Stories - SkinCare Advisor Chat

Функциональные требования в формате User Stories (для Части 1 проекта).

---

## US-1: Начало консультации

**Роль**: Как пользователь с проблемами кожи

**Действие**: Я хочу начать диалог с AI-агентом, отправив первое сообщение

**Цель**: Чтобы получить персональные рекомендации по уходу

**Критерии приемки**:

- Пользователь видит приветственное сообщение
- Может ввести и отправить сообщение
- Получает ответ от AI-агента

---

## US-2: Уточняющие вопросы

**Роль**: Как пользователь

**Действие**: Я хочу получать уточняющие вопросы о типе кожи

**Цель**: Чтобы агент собрал информацию для точных рекомендаций

**Критерии приемки**:

- Агент задает вопросы о типе кожи, проблемах, возрасте
- Вопросы понятны и естественны

---

## US-3: Получение рекомендаций

**Роль**: Как пользователь

**Действие**: Я хочу получить структурированные советы

**Цель**: Чтобы понять, какой уход мне подходит

**Критерии приемки**:

- Советы структурированы (утро/вечер)
- Указаны рекомендуемые ингредиенты
- Основано на базе знаний

---

## US-4: Просмотр истории

**Роль**: Как пользователь

**Действие**: Я хочу прокручивать историю диалога

**Цель**: Чтобы перечитать советы

**Критерии приемки**:

- История сохраняется в сессии
- Можно прокручивать вверх/вниз

---

## US-5: Мобильная версия

**Роль**: Как пользователь со смартфона

**Действие**: Я хочу использовать приложение на телефоне

**Цель**: Чтобы получать консультации в любом месте

**Критерии приемки**:

- Интерфейс адаптивный
- Читаемо на маленьком экране

---

## US-6: Обработка ошибок

**Роль**: Как пользователь

**Действие**: Я хочу видеть понятные сообщения об ошибках

**Цель**: Чтобы понять, как исправить проблему

**Критерии приемки**:

- Показываются понятные сообщения об ошибках
- Приложение остается работоспособным
